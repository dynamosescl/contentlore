FiveM Enhanced

Tracking the FiveM Enhanced platform transition across the GTA RP ecosystem. Framework compatibility, server migration status, and the technical changelog driving the rollout.
Intelligence Briefing · Updated 2026-04-28
FiveM Enhanced is the platform-level refactor Cfx.re started rolling out after Rockstar's acquisition closed and the Creator Platform Licence Agreement was reissued (12 Jan 2026). It is not a clean break — Enhanced is a runtime upgrade pushed to artifacts, not a parallel fork. Servers receive it the moment they update their server artifact past the cut-line build, and most of the visible changes for players land server-side. ContentLore tracks each tracked server's posture: have they updated artifacts, are scripts still loading clean, has the server owner said anything publicly. Expect "Monitoring" against most servers until owners publish migration notes.

Framework Compatibility ESX · QBCore · QBOX
Three frameworks dominate UK RP: ESX (legacy, biggest install base), QBCore (current default for newer launches), and QBOX (the modernised QBCore fork). Enhanced doesn't replace frameworks — it changes the runtime they sit on. Status reflects whether the framework's mainline branch loads cleanly against current Enhanced server artifacts.

ESX

Extended Survival Multiplayer · Legacy Stack
TestingLast checked 2026-04-28
ESX legacy builds run, but the older `esx_legacy` and `essentialmode` shim chains throw deprecation warnings under Enhanced's stricter scheduler. Active community forks (esx-framework / overextended) track Enhanced and patch ahead of artifact bumps. Tracked servers on bare ESX should expect a maintenance window when they update.

QBCore

Mainline · Most Common UK Framework
ReadyLast checked 2026-04-28
Mainline QBCore loads against Enhanced artifacts without intervention. Most UK RP launches in the last 24 months ship QBCore by default. The biggest risk is third-party resources written against pre-Enhanced natives — those need owner-side audits regardless of framework.

QBOX

Community Fork · Modernised QBCore
ReadyLast checked 2026-04-28
QBOX maintains an Enhanced-compatible release line and tags artifact-pinned versions on GitHub. Servers running QBOX upgrade in lockstep with the project's release cadence and report the smoothest Enhanced uptake of the three.

Technical Changelog Enhanced rollout
2026-01-12

Cfx Creator Platform Licence reissued

Cfx.re published the updated Creator Platform Licence Agreement post-Rockstar acquisition. The licence formalises Enhanced as the supported runtime path and reaffirms that third-party server software remains permitted under the agreement. Server owners had ~30 days to acknowledge in their server portals.

Source · Cfx.re forum announcement
2026-Q1

OneSync Infinity defaulted on new artifacts

Enhanced artifacts ship with OneSync Infinity enabled by default; legacy slot caps (32/64) require explicit opt-out. Tracked servers running the standard 64+ slot configuration are unaffected; smaller community servers may need to update server.cfg to suppress the default warning banner.

Source · Cfx.re changelog notes
2026-Q1

Stricter scheduler for resource startup

Enhanced enforces tighter timeouts on resource manifest parsing and `Citizen.CreateThread` warmup. Resources that depended on async race-conditions during start now log structured warnings. ESX legacy stacks see this most.

Source · artifact release notes
2026-03-15

FiveM hits 202k concurrent on Steam

FiveM (now first-party Cfx.re/Rockstar) hit a record 202k concurrent Steam users — a useful baseline for measuring scene impact when GTA 6 ships. The number reflects players actively connected to FiveM servers globally; the UK scene is a slice of that.

Source · Steam concurrent player tracker

Creator Impact

What changes for streamers

Mostly nothing visible. Performance and stability tend to improve. The risk window is narrow: a server's first night on a fresh Enhanced artifact is when surprise script breakages surface, and that's where streamers see "server down for maintenance" mid-RP.

Risk by stream type

Action-heavy / vehicle / shooter RP exposes Enhanced-related script issues fastest (collision sync, network entity handling). Slow-burn dialogue RP is the least exposed. Servers usually patch within hours of the first major report.

Income volatility

Creators with diversified platforms (Twitch + Kick + clips) absorb a server's bad night easily. Single-server, single-platform creators feel any Enhanced-related downtime more sharply. Most of the curated 26 are diversified.

How we'll track migrations

Three signals: (1) public statements from server owners, (2) stream-title keywords like "patch", "maintenance", "down" against a tracked server, (3) extended gaps in scene_snapshots for a server that's normally active. ContentLore surfaces signals here as they accumulate.
